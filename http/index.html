Protocolo HTTP

El protocolo HTTP (Hypertext Transfer Protocol) es un protocolo usado para transmitir documentos hipermedia estableciendo una comunicación entre navegadores web y servidores web, por lo tanto, sigue el modelo cliente-servidor.

Cliente-Servidor

Los clientes y servidores se comunican utilizando mensajes individuales, que si provienen del navegador se llaman peticiones y si originan del servidor se llaman respuestas.

Mensajes de HTTP

Como se mencionó anteriormente, existen dos tipos de mensajes HTTP: peticiones y respuestas, las cuales cuentan con una estructura propia.

Peticiones

Respuestas

Por diseño, HTTP no mantiene un estado, es decir: No hay ninguna relación entre alguna serie de peticiones enviadas durante una misma conexión. Si se requiere mantener un estado o un contexto para distintas peticiones, se pueden utilizar Cookies de HTTP.

Cookies HTTP

Las cookies son una pieza pequeña de datos que un servidor envía al navegador, quien puede elegir almacenarla y utilizarla en futuras peticiones al servidor. Usualmente son utilizadas para implementar autenticación o averiguar si distintas peticiones provienen del mismo navegador.

Recientemente, las cookies se utilizan por distintos sitios para personalizar contenido de acuerdo al usuario o para mantener un rastreo y análisis del usuario y de cómo interactúa con el sitio web.
